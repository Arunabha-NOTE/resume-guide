Skills

Skills should be organized into clear categories and listed in order of proficiency.

Focus on technical skills that are relevant to the position you're applying for, and consider grouping them by category.

Names of the sections don't matter you can tailor them as per your skillset. Mention only relevant things dont add things like data preparation, data cleaning, etc (I have seen someone do it and get roasted).

You can include relevant courseworks after this section but i recommend you don't.

Example Format:

Languages: Python, TypeScript, JavaScript, Java
Frameworks: Angular, React, Expo, FastAPI, Spring Boot, NestJs
DevOps: Docker, Kubernetes, CircleCI, GitHub Actions
Cloud: AWS, Azure
Databases: MongoDB, PostgreSQL

Do's and Don'ts

Do's

  • List skills in order of proficiency and relevance to the job.
  • Group skills into clear categories (e.g., Languages, Frameworks, Tools).
  • Add skills that complement your projects and experiences.
  • Also ensure your skills and projects are aligned to the job description.

Don'ts

  • Don't include basic skills like "Microsoft Word" please.
  • Avoid listing too many skills; focus on the most relevant ones.
  • Don't list skills you are not proficient in or haven't used recently (Don't include thinking i will do it later).
  • Avoid vague terms like "familiar with" or "exposure to" just write the skills.
  • Avoid a tools section where you write VSCode, IntelliJ, etc. Especially to the ones who write linux as a tool.
